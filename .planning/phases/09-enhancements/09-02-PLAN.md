## Critical Architecture Notes (Read Before Touching Any File)

### Token logging is split across TWO places — both must be updated

**Place 1: app/Models/OllamaProxy.php — logUsage() at line 270-284**
This is the PRIMARY logger. Called at line 177 (non-streaming) and line 151 (streaming).
For non-streaming, body['prompt_eval_count'] and body['eval_count'] are available at lines 167-170
BEFORE response()->json() is called. This is where prompt_tokens and completion_tokens are captured.

**Place 2: app/Services/CreditService.php — deductCredits() at line 52-98**
Called from ChatCompletionsController::store() line 147. Creates a SECOND UsageLog entry inline
at line 74-81. This second log also needs prompt_tokens / completion_tokens passed through.

### The estimateTokens() bug in ChatCompletionsController (line 557-576)
The current estimateTokens() ADDS the char-estimate ON TOP of prompt_eval_count + eval_count —
triple-counting tokens. When updating, use ONLY prompt_eval_count and eval_count. Remove
estimateTokens() from the controller entirely and replace inline.

### OllamaProxy non-streaming response format
The proxy calls Ollama at /api/chat and returns raw Ollama JSON via response()->json($body).
The Ollama non-streaming response includes:
  prompt_eval_count: 128   (input tokens)
  eval_count: 64           (output/completion tokens)
Both fields survive in the response body that ChatCompletionsController receives and decodes.

### OllamaProxy streaming path
The proxy accumulates totalTokens across chunks (lines 137-143) but does NOT track the split
per-chunk. For the streaming path: pass promptTokens=null, completionTokens=null to logUsage().
The blended rate in CostService will handle these rows. Streaming split is a deferred enhancement.

### CostService conversion rates (locked from CONTEXT.md)
- 1 credit = 0.001 USD
- USD to KWD: use constant 0.306 (1 USD approx 3.27 KWD)
- Blended rate for NULL-split rows: 70% input / 30% output of total tokens

### Public model pricing USD per 1M tokens (locked from CONTEXT.md)
- GPT-3.5 Turbo: input 0.50, output 1.50
- GPT-4:         input 3.00, output 6.00
- GPT-4o:        input 5.00, output 15.00  (PRIMARY BENCHMARK)
- Claude 3.5:    input 3.00, output 15.00
- Gemini 1.5 Pro: input 3.50, output 10.50

### Dashboard design (no Tailwind, pure CSS with CSS vars)
Current order: stats-grid -> model-catalog -> section-grid (API keys + top-up) -> recent usage
Savings card goes BETWEEN stats-grid and model-catalog.
Current usage table thead: Model | Tokens | Credits Used | Time
New usage table thead: Model | Input | Output | Tokens | vs GPT-4o | Credits Used | Time
CSS vars: --gold, --bg-card, --bg-secondary, --border, --text-muted, --text-primary
Green for savings: #28a745
All new CSS in @push('styles') block — no inline styles.

Method calculatePublicCost(int $promptTokens, int $completionTokens, string $publicModel): float
  Look up rates by $publicModel key. Return 0.0 if key not found.
  inputCost  = ($promptTokens / 1_000_000) * rate['input']
  outputCost = ($completionTokens / 1_000_000) * rate['output']
  Return round(inputCost + outputCost, 6)
  Example: calculatePublicCost(1000, 500, 'gpt-4o') = (1000/1M * 5.00) + (500/1M * 15.00) = 0.0125

Method calculatePublicCostBlended(int $totalTokens, string $publicModel): float
  promptTokens     = (int) round($totalTokens * self::BLENDED_INPUT_RATIO)
  completionTokens = (int) round($totalTokens * self::BLENDED_OUTPUT_RATIO)
  Return calculatePublicCost($promptTokens, $completionTokens, $publicModel)

Method creditsToUsd(int $credits): float
  Return round($credits * self::CREDIT_TO_USD, 6)

Method usdToKwd(float $usd): float
  Return round($usd * self::USD_TO_KWD, 4)

Method getMonthlySavings($user): array
  Query: UsageLog where user_id=$user->id AND created_at >= now()->startOfMonth()
  Select: credits_deducted, tokens_used, prompt_tokens, completion_tokens
  If empty, return emptySavingsResult().

  Initialize: $ourCostUsd = 0.0, $publicTotals keyed by model key all 0.0, $hasAnyEstimate = false
  Loop logs:
    $ourCostUsd += creditsToUsd((int)log.credits_deducted)
    $isBlended = is_null(log.prompt_tokens) || is_null(log.completion_tokens)
    if ($isBlended) $hasAnyEstimate = true
    foreach rates key:
      if blended: publicTotals[key] += calculatePublicCostBlended(tokens_used, key)
      else: publicTotals[key] += calculatePublicCost(prompt_tokens, completion_tokens, key)

  Build $comparisons array. For each publicTotals entry:
    $savingsUsd = max(0.0, $publicCostUsd - $ourCostUsd)
    $entry = [model=>key, name=>rates[key][name], public_cost_usd=>round(4dp),
              public_cost_kwd=>usdToKwd(), savings_usd=>round(4dp),
              savings_kwd=>usdToKwd(savingsUsd), is_estimate=>$hasAnyEstimate]
    if key == 'gpt-4o': $benchmark = $entry

  Return [our_cost_usd=>round(4dp), our_cost_kwd=>usdToKwd(ourCostUsd),
          comparisons=>$comparisons, benchmark=>$benchmark, total_calls=>$logs->count()]

Method getRecentCallComparisons($user, int $limit = 10): array
  Query: UsageLog where user_id=$user->id, orderByDesc('created_at'), take($limit)
