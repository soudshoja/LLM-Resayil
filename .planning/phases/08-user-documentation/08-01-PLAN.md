**`docs/api-reference.md`**

Write a full API Reference covering both endpoints.

**Base URL:** `https://llm.resayil.io/api/v1`

**Endpoint 1 — POST /api/v1/chat/completions**

- Description: Send a chat message and receive an LLM response. OpenAI-compatible.
- Authentication: `Authorization: Bearer YOUR_API_KEY`
- Content-Type: `application/json`
- Request body fields (with types and descriptions):
  - `model` (string, required) — model ID from GET /api/v1/models
  - `messages` (array, required) — array of `{role, content}` objects; roles: `system`, `user`, `assistant`
  - `stream` (boolean, optional, default false) — if true, response is Server-Sent Events stream
  - `temperature` (number, optional, 0–2) — sampling temperature
  - `max_tokens` (integer, optional) — maximum tokens to generate
- Successful response (non-streaming): show full JSON example with `id`, `object`, `created`, `model`, `choices[].message.content`, `usage.prompt_tokens`, `usage.completion_tokens`, `usage.total_tokens`
- Successful response (streaming): show first few SSE data lines including `[DONE]`
- Error responses: 401, 402, 429, 503 (refer to Error Codes page)

Include a complete curl example for non-streaming and a note about streaming.

**Endpoint 2 — GET /api/v1/models**

- Description: List all available models.
- Authentication: `Authorization: Bearer YOUR_API_KEY`
- Response: JSON object with `data` array; each item has `id`, `object: "model"`, `created`, `owned_by`
- Show a sample truncated response with 2–3 models

**`docs/models.md`**

**`docs/error-codes.md`**

Write an Error Codes reference covering all error HTTP status codes:

For each code, provide: HTTP status, when it occurs, example JSON response body, how to resolve.

- **401 Unauthorized** — API key missing from request or key does not exist / has been revoked. Resolution: check `Authorization: Bearer` header; create a new key at /api-keys
- **402 Payment Required** — Account has insufficient credits to complete the request. Resolution: purchase a top-up at /billing/plans
- **429 Too Many Requests** — Rate limit for the current tier exceeded. Resolution: slow down requests; upgrade plan for higher limits
- **503 Service Unavailable** — The requested model is currently unavailable (GPU offline, model not loaded). Resolution: retry after a short delay; try a different model

Also add a general note: all error responses follow the format `{"error": {"code": "...", "message": "..."}}`.
