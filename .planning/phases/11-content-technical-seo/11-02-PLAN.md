<objective>
Implement hreflang tags on all 20+ pages (EN and AR versions) to help search engines understand language/regional versions exist, improve crawl efficiency, and prevent duplicate content penalties.

Purpose: Signal to Google that AR and EN pages are alternate versions, prevent SEO dilution across language variants, improve ranking for language-specific searches, enable users to switch languages with proper URL structure.

Output: Hreflang link tags in <head> of every user-facing page, LocaleController handling language switching, route definitions for locale switching.
</objective>

Hreflang implementation must account for:
1. Current locale prefix structure (/locale/en, /locale/ar routes)
2. 20+ distinct pages across marketing, auth, dashboard, and docs sections
3. Google's hreflang specification (mutual annotation, absolute URLs, correct lang codes)
4. Blade template structure (use @include or component for reusability)

SEO best practice context:
- hreflang uses ISO 639-1 codes: en-US or en, ar or ar-SA
- Absolute URLs required (not relative paths)
- Mutual annotation: EN page links to AR, AR page links to EN
- x-default hreflang for default version (typically English)
- Self-referential hreflang optional but helps crawlers

  <name>Task 1: Create hreflang Blade component for reusable template usage</name>
  <files>resources/views/components/hreflang.blade.php, routes/web.php</files>
  <action>
Create a reusable Blade component at resources/views/components/hreflang.blade.php that generates hreflang link tags:

Component signature:
```blade
@props([
  'currentPath' => request()->path(),
  'locale' => app()->getLocale(),
  'isXDefault' => false,  // true for landing page to set x-default
])
```

Component logic:
1. Determine the current route/page path
2. Generate English version URL: route or URL with locale prefix
3. Generate Arabic version URL: append ?lang=ar or /ar variant
4. Output link tags in <head>:
   ```html
   <link rel="alternate" hreflang="en" href="https://llm.resayil.io/dashboard" />
   <link rel="alternate" hreflang="ar" href="https://llm.resayil.io/dashboard?lang=ar" />
   <link rel="alternate" hreflang="x-default" href="https://llm.resayil.io/dashboard" />
   ```

Key features:
- Uses absolute URLs: url() or env('APP_URL')
- Detects current locale: app()->getLocale()
- Adds self-referential hreflang (page links to itself)
- Supports x-default variant for landing pages (isXDefault prop)
- Handles both / and named routes (routes/web.php may use named routes)

Fallback logic:
- If route() fails, use url() with path
- If locale not detected, default to 'en'

Component structure:
```blade
@php
  $enUrl = url()->to(str_replace('/ar', '', $currentPath));
  $arUrl = str_contains($currentPath, '/ar')
    ? url()->to($currentPath)
    : url()->to($currentPath . (str_contains($currentPath, '?') ? '&lang=ar' : '?lang=ar'));

  $currentUrl = url()->current();
@endphp

<link rel="alternate" hreflang="en" href="{{ $enUrl }}" />
<link rel="alternate" hreflang="ar" href="{{ $arUrl }}" />
@if ($isXDefault)
  <link rel="alternate" hreflang="x-default" href="{{ $enUrl }}" />
@endif
<link rel="alternate" hreflang="{{ app()->getLocale() }}" href="{{ $currentUrl }}" />
```

Avoid: Hardcoding URLs (use helpers), complex regex patterns (keep it simple).
