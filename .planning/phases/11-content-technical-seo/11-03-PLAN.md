<objective>
Add semantic alt text to 50+ images across marketing pages, docs pages, and dashboard pages to improve SEO image crawlability, accessibility for screen readers, and provide context for images in search results.

Purpose: Help search engines understand image content and context, improve page ranking for image search, provide fallback text when images fail to load, enhance accessibility for users with visual impairments.

Output: All images across 8+ pages have semantic alt text (120+ characters max), properly structured HTML, and descriptive context.
</objective>

Image alt text guidelines from SEO best practices:
- Semantic: Describe what the image IS and DOES, not "image of..." or "screenshot of..."
- Contextual: Include relevant keywords but avoid keyword stuffing
- Length: 120-125 characters max (Google truncates longer)
- Structure: [Object/Action] + [Context/Benefit] (e.g., "Dashboard showing real-time API usage statistics")
- Accuracy: Must truthfully describe what the image shows
- Redundancy: Don't repeat text already present in page (e.g., if page says "Cost Calculator", don't say "Cost Calculator tool")

Accessibility standards (WCAG 2.1 AA):
- All non-decorative images must have alt text
- Decorative images can have empty alt="" but must be marked role="presentation"
- Complex images (charts, diagrams) benefit from longer description

Image audit findings (from /seo skill analysis):
- Landing pages: 8-12 images (hero, features, testimonials, pricing, benefits, CTAs)
- Docs pages: 4-6 images (code examples, architecture diagrams, UI screenshots)
- Marketing pages: 8-10 images per page (cost-calculator, comparison, alternatives, dedicated-server)
- Dashboard: 3-5 images (charts, icons, status indicators)
- Total: 50-80 images requiring alt text
